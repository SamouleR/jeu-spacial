🚀 Présentation et Accès

SpaceWar est un jeu de stratégie spatiale tactique au tour par tour. Il met en scène un affrontement entre deux joueurs sur une grille de 10x10 cases. Le projet a été réalisé en utilisant PHP Orienté Objet pour la logique serveur et JavaScript (AJAX/jQuery) pour l'interactivité client.

📜 Règles Générales

  • Objectif : Détruire le vaisseau adverse en réduisant ses PV à 0.
  • Tour par Tour : Les joueurs jouent chacun leur tour. Un système de synchronisation empêche de jouer deux fois de suite.
  • Gestion de l'Énergie : Chaque action consomme de l'énergie (Déplacement, Tir, Drones). L'énergie se régénère de 10 pts/tour.
  • Déplacements : Le coût en énergie dépend de la distance parcourue (2 énergie / case).

🤖 Guide Tactique des Drones

Chaque drone a un comportement codé spécifiquement dans la classe Partie.php. Voici leurs effets techniques :

⚔️ Drone d'Attaque
Coût : 30 Énergie
Portée lancement : 4 cases

Immédiat : Inflige 15 dégâts directs au vaisseau ennemi.
Passif (Fin de tour) : Si l'ennemi reste à 3 cases ou moins du drone, il subit automatiquement 10 dégâts supplémentaires à chaque tour.

🛡️ Drone Bouclier
Coût : 30 Énergie
Position : Sur soi

Ajoute +30 Points de Bouclier au joueur. Les points de bouclier absorbent les dégâts reçus avant que la coque (PV) ne soit touchée.

⚡ Drone Sabotage
Coût : 30 Énergie
Portée : 3 cases

Pirate le système d'armement adverse. Applique un Malus pour 2 tours.
Effet : Les dégâts infligés par l'ennemi sont divisés par 2.

🔧 Drone Réparation
Coût : 30 Énergie
Position : Sur soi

Se soude à la coque pour effectuer une réparation d'urgence.
Restitue immédiatement +20 PV au vaisseau.

💥 Drone Kamikaze
Coût : 30 Énergie
Portée : 1 case (Adjacent)

Le drone se jette sur l'ennemi et explose.
Effet Critique : Inflige 30 dégâts massifs. Le drone est détruit instantanément.

Rôle des Composants

  • Modèle (PHP) : Les classes Vaisseau, Drone et Partie encapsulent toute la logique métier et les règles (vérification de portée, coûts, dégâts).
  • Contrôleur (action.php) : Il agit comme une passerelle. Il charge l'état depuis le fichier, exécute la demande du client, sauvegarde et renvoie la réponse.
  • Vue (HTML/JS) : board.js et les fichiers HTML gèrent uniquement l'affichage et l'envoi des commandes. Ils ne calculent rien (pour éviter la triche).

⚙️ Mécanismes Complexes

1. Synchronisation par Polling

Le jeu utilise le Polling AJAX. Le client JavaScript interroge le serveur toutes les 2 secondes via get_status. Le serveur renvoie un objet JSON complet contenant les positions, les PV et l'ID du joueur actif. Cela permet une mise à jour quasi temps-réel de l'interface sans rechargement de page.

2. Prévisualisation Visuelle (UX)

Pour améliorer l'expérience, un système de prévisualisation a été codé en JS. Lors de la saisie de coordonnées, le script utilise les sélecteurs d'attributs td[data-x="..."][data-y="..."] pour cibler la case correspondante et lui appliquer une classe CSS spécifique (.preview-move), affichant un cadre rose fluo par-dessus la grille.

3. Système de Sabotage et Malus

Le drone de sabotage applique un "Malus" temporaire. Cette donnée est gérée par la classe Joueur et transmise dans le flux JSON. Le frontend détecte ce malus et ajoute dynamiquement une classe CSS animée .cell-sabotaged (clignotement orange) sur le vaisseau touché, ainsi qu'un message d'alerte dans le panneau de contrôle.

📊 Bilan du Projet

  • Objectifs atteints : Le jeu est fonctionnel, multijoueur (simulé sur deux fenêtres), et respecte strictement les contraintes POO/AJAX.
  • Qualité visuelle : Un gros effort a été fourni sur le design (Layout Flexbox réactif, thème néon cohérent, animations CSS) pour une immersion totale.
  • Robustesse : La logique critique est entièrement côté serveur (PHP), rendant le jeu sécurisé contre la modification simple du DOM.
  • Améliorations possibles : Utilisation de WebSockets pour remplacer le Polling (plus performant), ou ajout d'une base de données SQL pour gérer des sessions multiples.

PHP Partie.php : Gestion du tour & Ravitaillement 

public function terminerTour(): void {
    // Gestion Malus (Décrémentation)
    $joueurCourant = $this->getJoueurActif();
    if ($joueurCourant->malusTirTours > 0) $joueurCourant->malusTirTours--;

    // Changement de joueur
    $this->joueurActifId = $this->joueurActifId === 1 ? 2 : 1;
    $this->tourCourant++;

    // Régénération Énergie
    $this->getJoueurActif()->vaisseau->regenererEnergie(10);

    // Ravitaillement drones (tous les 5 tours)
    // On utilise le modulo pour détecter le 5ème tour
    if ($this->tourCourant > 1 && $this->tourCourant % 5 === 1) {
        $this->joueur1->nbDrones++;
        $this->joueur2->nbDrones++;
    }
}

            $drone->consommer(10); // Consommation pour maintien
            
            // Logique Passive du Drone Attaque
            if ($drone->getType() === 'attaque') {
                $adversaire = ($joueur->id === 1) ? $this->joueur2 : $this->joueur1;
                $distance = $this->calculerDistance($drone->getPosition(), $adversaire->vaisseau->getPosition());

                if ($distance <= 3) {
                    $adversaire->vaisseau->subirDegats(10);
                }
            }
        }
    }
