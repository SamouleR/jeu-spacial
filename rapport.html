🚀 Présentation et Accès

SpaceWar est un jeu de stratégie spatiale tactique au tour par tour. Il met en scène un affrontement entre deux joueurs sur une grille de 10x10 cases. Le projet a été réalisé en utilisant PHP Orienté Objet pour la logique serveur et JavaScript (AJAX/jQuery) pour l'interactivité client.

📜 Règles du Jeu

  • Objectif : Détruire le vaisseau adverse en réduisant ses PV à 0.
  • Tour par Tour : Les joueurs jouent chacun leur tour. Un système de synchronisation empêche de jouer deux fois de suite.
  • Gestion de l'Énergie : Chaque action consomme de l'énergie (Déplacement, Tir, Drones). L'énergie se régénère de 10 pts/tour.
  • Déplacements : Le coût en énergie dépend de la distance parcourue (2 énergie / case).
  • Arsenal :
    • Canon Principal : 10 Énergie. Portée 4 cases.
    • Drones (Stock limité) : Coût 30 Énergie. Types variés (Attaque, Bouclier, Sabotage, Réparation, Kamikaze).
  • Ravitaillement : Le stock de drones augmente de +1 tous les 5 tours.

🏗️ Organisation du Code

Le projet respecte une séparation stricte entre le modèle de données, le contrôleur d'actions et la vue.

1. Modèle (Backend PHP)

Les entités sont définies dans des classes PHP distinctes :

  • Vaisseau.php : Gère les attributs vitaux (Vie, Énergie, Position) et la logique de coût des actions.
  • Drone.php : Définit les types de drones et leur comportement.
  • Joueur.php : Encapsule le vaisseau et l'inventaire de drones d'un joueur.
  • Partie.php : Classe maîtresse. Elle contient l'état global du jeu, gère les tours, valide les actions et applique les effets de fin de tour.

2. API & Persistance

  • action.php : Agit comme une API JSON. Il reçoit les requêtes AJAX, instancie la classe Partie (chargée depuis un fichier sérialisé partie.dat), exécute l'action et renvoie le nouvel état.

3. Vue (Frontend)

  • joueur1.html / joueur2.html : Structure HTML.
  • style.css : Design unique "Cyberpunk" utilisant Flexbox pour le layout.
  • board.js : Script JS chargé de générer la grille HTML dynamiquement à partir de l'état JSON reçu.

⚙️ Mécanismes Complexes

1. Synchronisation par Polling

Comme le protocole HTTP est sans état, le client ne sait pas quand l'autre joueur a joué. Nous utilisons un setInterval en JavaScript qui interroge action.php?type_action=get_status toutes les 2 secondes. Cela permet de mettre à jour l'interface (PV, position, tour actuel) en quasi temps réel sans recharger la page.

2. Prévisualisation Visuelle (UX)

Pour améliorer l'expérience utilisateur, nous avons développé un système de prévisualisation en JavaScript. Lorsque le joueur entre des coordonnées ciblées (X, Y), le script cherche la case correspondante dans le DOM (grâce aux attributs data-x et data-y injectés par board.js) et lui applique une classe CSS .preview-move (cadre rose fluo).

3. Système de Sabotage et Malus

Le drone de sabotage applique un "Malus" temporaire. Cette information est stockée côté serveur. Lors de l'envoi de l'état JSON, le serveur inclut le champ malus. Le client JS lit cette valeur et applique dynamiquement la classe CSS animée .cell-sabotaged sur le vaisseau concerné, le faisant clignoter en orange sur la grille.

📊 Bilan du Projet

  • Objectifs atteints : Le jeu est fonctionnel, multijoueur (simulé), et respecte les contraintes POO/AJAX.
  • Qualité visuelle : Un effort particulier a été mis sur le design (CSS Grid/Flexbox, animations CSS, thème néon) pour l'immersion.
  • Robustesse : La vérification des règles (coûts en énergie, portée des tirs, limites de la carte) est effectuée côté serveur (PHP) pour éviter la triche.
  • Améliorations futures : On pourrait ajouter une base de données MySQL pour gérer plusieurs parties en parallèle, ou des bruitages sonores lors des tirs.

PHP Vaisseau.php 

<?php
class Vaisseau {
    private string $nom;
    private int $vie;
    private int $energie;
    private int $puissanceTir;
    private array $position;

    const COUT_PAR_CASE = 2;
    const COUT_TIR = 10;
    const COUT_DRONE = 30;

    public function __construct(string $nom, int $vieMax, int $energieMax, int $puissanceTir, array $position) {
        $this->nom = $nom;
        $this->vie = $vieMax;
        $this->energie = $energieMax;
        $this->puissanceTir = $puissanceTir;
        $this->position = $position;
    }

    public function deplacer(int $x, int $y): bool {
        if ($this->estDetruit()) return false;
        if ($x < 1 || $x > 10 || $y < 1 || $y > 10) return false;

        $distance = abs($x - $this->position['x']) + abs($y - $this->position['y']);
        $coutTotal = $distance * self::COUT_PAR_CASE;

        if ($this->energie < $coutTotal) return false;

        $this->energie -= $coutTotal;
        $this->position = ['x' => $x, 'y' => $y];
        return true;
    }

    // ... autres méthodes (tirer, lancerDrone, getters) ...
}

PHP action.php 

<?php
header('Content-Type: application/json; charset=utf-8');
require_once 'Partie.php';
define('PARTIE_FILE', __DIR__ . '/partie.dat');

function chargerPartie(): Partie {
    if (file_exists(PARTIE_FILE)) {
        $data = file_get_contents(PARTIE_FILE);
        $obj = @unserialize($data);
        if ($obj instanceof Partie) return $obj;
    }
    return new Partie(1);
}

function sauvegarderPartie(Partie $partie): void {
    file_put_contents(PARTIE_FILE, serialize($partie));
}

$joueurId = isset($_POST['joueur_id']) ? (int)$_POST['joueur_id'] : 1;
$typeAction = $_POST['type_action'] ?? 'get_status';

$partie = chargerPartie();

if ($typeAction === 'initialiser') {
    $partie = new Partie(1);
}

$result = $partie->executerAction($_POST, $joueurId);
sauvegarderPartie($partie);

echo json_encode([
    'action_result' => $result,
    'etat_jeu' => $partie->getEtatPourClient()
]);
exit;
?>

const GRID_SIZE = 10;
